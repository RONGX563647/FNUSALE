## 二、安全性问题与优化建议

### 2.1 接口限流缺失（严重）

**问题描述：**

* 所有接口无限流保护，容易被恶意刷接口

* 注册、登录、发送验证码等接口容易被滥用

* 可能导致短信/邮件费用损失和服务器资源耗尽

**风险场景：**

1. 恶意用户循环调用 `/user/captcha/send` 消耗短信配额
2. 暴力破解密码（无失败次数限制）
3. DDoS 攻击导致服务不可用

**解决方案：**

#### 方案 1：Redis 限流（推荐）

```java
@Component
public class RateLimiterInterceptor implements HandlerInterceptor {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
        String uri = request.getRequestURI();
        String ip = getClientIp(request);
        
        // 发送验证码接口：每 IP 每天最多 10 次
        if (uri.contains("/captcha/send")) {
            String key = "rate:limit:captcha:" + ip + ":" + LocalDate.now();
            Long count = redisTemplate.opsForValue().increment(key);
            if (count == 1) {
                redisTemplate.expire(key, 1, TimeUnit.DAYS);
            }
            if (count > 10) {
                throw new BusinessException("发送次数过多，请明天再试");
            }
        }
        
        // 登录接口：每 IP 每分钟最多 5 次
        if (uri.contains("/login")) {
            String key = "rate:limit:login:" + ip + ":" + System.currentTimeMillis() / 60000;
            Long count = redisTemplate.opsForValue().increment(key);
            if (count == 1) {
                redisTemplate.expire(key, 1, TimeUnit.MINUTES);
            }
            if (count > 5) {
                throw new BusinessException("登录次数过多，请稍后再试");
            }
        }
        
        return true;
    }
}
```

**问题描述：**

* 未要求密码复杂度（大小写字母 + 数字 + 特殊字符）

* 未实现密码历史记录（可重复使用旧密码）

**当前代码：**

```java
// UserValidator.java
public static void validatePassword(String password) {
    if (password == null || password.length() < 6 || password.length() > 20) {
        throw new BusinessException("密码长度必须在 6-20 位之间");
    }
}
```

**优化建议：**

```java
public static void validatePassword(String password) {
    if (password == null || password.length() < 8 || password.length() > 20) {
        throw new BusinessException("密码长度必须在 8-20 位之间");
    }
    
    // 至少包含大写字母、小写字母、数字、特殊字符中的 3 种
    int count = 0;
    if (password.matches(".*[A-Z].*")) count++;
    if (password.matches(".*[a-z].*")) count++;
    if (password.matches(".*\\d.*")) count++;
    if (password.matches(".*[!@#$%^&*(),.?\":{}|<>].*")) count++;
    
    if (count < 3) {
        throw new BusinessException("密码必须包含大写字母、小写字母、数字、特殊字符中的至少 3 种");
    }
}
```

### 2.3 登录失败次数限制缺失（严重）

**问题描述：**

* 登录接口无失败次数限制

* 容易被暴力破解或撞库攻击

**解决方案：**

```java
@Service
public class LoginAttemptService {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    private static final int MAX_ATTEMPTS = 5;
    private static final long LOCK_TIME_MINUTES = 15;
    
    public void recordLoginAttempt(String account) {
        String key = "login:attempts:" + account;
        Long attempts = redisTemplate.opsForValue().increment(key);
        if (attempts == 1) {
            redisTemplate.expire(key, LOCK_TIME_MINUTES, TimeUnit.MINUTES);
        }
    }
    
    public void clearLoginAttempts(String account) {
        String key = "login:attempts:" + account;
        redisTemplate.delete(key);
    }
    
    public boolean isLocked(String account) {
        String key = "login:attempts:" + account;
        Long attempts = redisTemplate.opsForValue().get(key);
        return attempts != null && attempts >= MAX_ATTEMPTS;
    }
    
    public Long getRemainingAttempts(String account) {
        String key = "login:attempts:" + account;
        Long attempts = redisTemplate.opsForValue().get(key);
        if (attempts == null) {
            return (long) MAX_ATTEMPTS;
        }
        return Math.max(0, MAX_ATTEMPTS - attempts);
    }
}

// 在登录方法中使用
public LoginVO login(UserLoginDTO dto) {
    // 检查是否被锁定
    if (loginAttemptService.isLocked(dto.getPhone() != null ? dto.getPhone() : dto.getEmail())) {
        throw new BusinessException("登录失败次数过多，请 15 分钟后再试");
    }
    
    try {
        // 原有登录逻辑...
        loginAttemptService.clearLoginAttempts(account);
        return loginVO;
    } catch (BusinessException e) {
        // 记录失败次数
        loginAttemptService.recordLoginAttempt(account);
        Long remaining = loginAttemptService.getRemainingAttempts(account);
        throw new BusinessException("密码错误，还剩" + remaining + "次机会");
    }
}
```

```java
// 滑动过期：每次使用 Token 时，如果剩余有效期不足 30 分钟，自动续期
public String refreshTokenIfNeeded(String token) {
    Claims claims = JwtUtil.parseToken(token);
    if (claims != null) {
        Date expiration = claims.getExpiration();
        long ttl = expiration.getTime() - System.currentTimeMillis();
        
        // 剩余有效期不足 30 分钟，自动续期
        if (ttl < 30 * 60 * 1000) {
            Long userId = JwtUtil.getUserId(token);
            String username = JwtUtil.getUsername(token);
            String identityType = JwtUtil.getIdentityType(token);
            return JwtUtil.generateAccessToken(userId, username, identityType);
        }
    }
    return token;
}
```

### 2.5 Spring Security 配置为开发模式（严重）

**问题描述：**

* 位置：`SecurityConfig.java`

* 现状：所有接口都允许访问，未启用任何安全保护

**当前代码：**

```java
// SecurityConfig.java:23-31
http
    // 禁用 CSRF
    .csrf(AbstractHttpConfigurer::disable)
    // 允许所有请求
    .authorizeHttpRequests(auth -> auth.anyRequest().permitAll())
    // 禁用表单登录
    .formLogin(AbstractHttpConfigurer::disable)
    // 禁用 HTTP Basic 认证
```

**生产环境配置：**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            // 生产环境启用 CSRF（前后端分离可使用 JWT 替代）
            .csrf(csrf -> csrf
                .ignoringRequestMatchers("/api/**")
                .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
            )
            // 授权配置
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/user/register/**", "/user/login/**", "/user/captcha/**").permitAll()
                .requestMatchers("/user/logout", "/user/refresh-token").permitAll()
                .requestMatchers("/user/info", "/user/my/**").authenticated()
                .requestMatchers("/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            )
            // 添加 JWT 认证过滤器
            .addFilterBefore(jwtAuthenticationFilter(), UsernamePasswordAuthenticationFilter.class)
            // 异常处理
            .exceptionHandling(ex -> ex
                .authenticationEntryPoint((request, response, authException) -> {
                    response.setStatus(401);
                    response.setContentType("application/json");
                    response.getWriter().write("{\"code\":401,\"message\":\"未授权\"}");
                })
                .accessDeniedHandler((request, response, accessDeniedException) -> {
                    response.setStatus(403);
                    response.setContentType("application/json");
                    response.getWriter().write("{\"code\":403,\"message\":\"拒绝访问\"}");
                })
            );
        
        return http.build();
    }
    
    @Bean
    public JwtAuthenticationFilter jwtAuthenticationFilter() {
        return new JwtAuthenticationFilter();
    }
}
```
