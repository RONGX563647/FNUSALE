### 步骤 3：添加异常处理和重试机制

**文件：** `EmailServiceImpl.java`

**修改内容：**
```java
// 添加重试配置
@Value("${spring.mail.send.retry.max:3}")
private int maxRetryAttempts;

@Value("${spring.mail.send.retry.delay:1000}")
private long retryDelayMs;

// 修改 sendVerificationCode 方法，添加重试逻辑
@Override
public void sendVerificationCode(String to, String captcha) {
    int retryCount = 0;
    while (retryCount < maxRetryAttempts) {
        try {
            MimeMessage message = mailSender.createMimeMessage();
            MimeMessageHelper helper = new MimeMessageHelper(message, true, "UTF-8");
            
            helper.setFrom(fromEmail);
            helper.setTo(to);
            helper.setSubject("【FNUSALE】验证码");
            
            String content = buildVerificationEmailContent(captcha);
            helper.setText(content, true);
            
            mailSender.send(message);
            log.info("验证码邮件发送成功 - 收件人：{}", to);
            return; // 发送成功，退出循环
        } catch (MessagingException e) {
            retryCount++;
            if (retryCount >= maxRetryAttempts) {
                log.error("验证码邮件发送失败（已达最大重试次数） - 收件人：{}, 错误：{}", to, e.getMessage(), e);
                throw new RuntimeException("验证码邮件发送失败，请稍后重试", e);
            }
            log.warn("验证码邮件发送失败，准备重试 {}/{} - 收件人：{}, 错误：{}", 
                retryCount, maxRetryAttempts, to, e.getMessage());
            
            try {
                Thread.sleep(retryDelayMs);
            } catch (InterruptedException ie) {
                Thread.currentThread().interrupt();
                throw new RuntimeException("发送被中断", ie);
            }
        }
    }
}
```

## 五、注意事项

### 5.1 163 邮箱配置说明

1. **授权码获取**
   - 登录 163 邮箱（https://mail.163.com）
   - 进入「设置」→「POP3/SMTP/IMAP」
   - 开启「SMTP 服务」
   - 点击「授权密码」生成授权码
   - 使用授权码而非登录密码

2. **端口选择**
   - 465 端口：SSL 加密（推荐）
   - 25 端口：非加密（很多运营商封端口，不推荐）

3. **发送频率限制**
   - 163 邮箱有发送频率限制
   - 建议单账号每天不超过 200 封邮件
   - 避免被识别为垃圾邮件

### 5.2 生产环境建议

1. **使用企业邮箱**
   - 建议使用企业邮箱（如阿里云企业邮箱）
   - 避免使用个人邮箱
   - 提高邮件送达率

2. **配置多个 SMTP 账号**
   - 支持负载均衡
   - 避免单一账号被限流

3. **使用邮件队列**
   - 使用 RocketMQ 或 RabbitMQ
   - 异步发送邮件
   - 提高系统吞吐量

4. **监控邮件发送**
   - 统计发送成功率
   - 监控发送失败率
   - 设置告警阈值

### 5.3 安全建议

1. **敏感信息保护**
   - 不要在日志中打印邮件内容
   - 不要在日志中打印授权码
   - 使用脱敏工具处理邮箱地址

2. **防止邮件滥用**
   - 限制单个邮箱的发送频率
   - 限制单个 IP 的发送频率
   - 使用验证码防止恶意请求

3. **邮件内容安全**
   - 不要在邮件中包含密码
   - 不要在邮件中包含敏感链接
   - 使用 HTTPS 链接（如果有）
