## 一、启动用户服务

### 1.1 启动服务

```bash
cd e:\CODE\git\FNUSALE\backend\fnusale-user
```

## 二、测试用例

### 测试 1：发送验证码到邮箱

**测试目标：** 验证能否正常发送验证码邮件

**测试步骤：**

1. 使用 Postman 或 curl 发送请求：

```bash
curl -X POST http://localhost:8101/user/captcha/send \
  -H "Content-Type: application/json" \
  -d '{
    "account": "lrx563647@yeah.net",
    "type": "LOGIN"
  }'
```

2. 检查返回结果：
```json
{
  "code": 200,
  "message": "验证码发送成功",
  "data": null
}
```

3. 检查日志输出：
```
验证码邮件发送成功 - 收件人：lrx563647@yeah.net
```

4. 检查邮箱收件箱：
- 登录邮箱：https://mail.163.com
- 查看收件箱
- 查找主题为「【FNUSALE】验证码」的邮件

**预期结果：**
- ✅ HTTP 状态码：200
- ✅ 返回消息："验证码发送成功"
- ✅ 日志输出："验证码邮件发送成功 - 收件人：lrx563647@yeah.net"
- ✅ 邮箱收到验证码邮件

**验证点：**
- ✅ 邮件主题正确：【FNUSALE】验证码
- ✅ 邮件内容包含验证码（6 位数字）
- ✅ 邮件样式正常（HTML 渲染）
- ✅ 提示信息完整（5 分钟有效期、安全提示）

## 四、常见问题排查

### 4.1 邮件发送失败

**问题：** 邮件发送失败，提示 "验证码邮件发送失败，请稍后重试"

**排查步骤：**

1. 检查 application.yml 配置：
```yaml
spring:
  mail:
    host: smtp.163.com
    port: 465
    username: lrx563647@yeah.net
    password: YEUdHnDgEtJNBLQd  # 确认是授权码，不是登录密码
```

2. 检查 163 邮箱设置：
- 登录 163 邮箱：https://mail.163.com
- 进入「设置」→「POP3/SMTP/IMAP」
- 确认「SMTP 服务」已开启
- 重新生成授权码

3. 检查网络连接：
```bash
telnet smtp.163.com 465
```

4. 检查防火墙设置：
- 确认 465 端口未被阻止
- 确认出站流量未被限制

### 4.2 邮件发送超时

**问题：** 邮件发送超时

**排查步骤：**

1. 检查网络连接：
```bash
ping smtp.163.com
```

2. 检查超时配置：
```yaml
spring:
  mail:
    properties:
      mail:
        smtp:
          connectiontimeout: 10000  # 10 秒
          timeout: 10000  # 10 秒
          writetimeout: 10000  # 10 秒
```

3. 检查代理设置：
- 确认没有代理拦截 SMTP 连接
- 确认 DNS 解析正常

### 4.3 邮件被识别为垃圾邮件

**问题：** 邮件被识别为垃圾邮件

**排查步骤：**

1. 检查邮件内容：
- 避免使用敏感词汇
- 避免使用大量链接
- 避免使用大量图片

2. 检查发送频率：
- 单个邮箱每天不超过 200 封邮件
- 避免短时间内大量发送

3. 配置 SPF 记录：
- 在域名 DNS 中添加 SPF 记录
- 提高邮件送达率
