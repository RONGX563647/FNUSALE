**版本**: v1.1.1

## 一、模块概述

### 1.1 模块定位

营销模块是FNUSALE校园二手交易平台的促活与转化核心模块，承担着提升用户活跃度、促进交易转化的关键职责。作为校园专属二手交易平台，营销模块的核心价值在于：

- **促活转化**：通过优惠券和秒杀活动，提升用户活跃度和交易转化率
- **场景贴合**：营销活动贴合校园场景（毕业季、开学季），符合师生使用习惯
- **轻量实用**：秒杀规模适配校园用户规模，无需复杂分布式架构

### 1.2 模块职责

| 职责领域 | 具体内容 |
|---------|---------|
| 优惠券管理 | 优惠券创建、发放、领取、核销、过期处理 |
| 秒杀活动 | 秒杀活动配置、库存管理、抢购处理、结果查询 |
| 活动提醒 | 秒杀提醒设置、提醒通知推送 |
| 数据统计 | 优惠券使用统计、秒杀参与统计、活动效果分析 |

### 1.3 模块边界

**包含功能**：
- 优惠券全生命周期管理（创建→发放→领取→使用→过期）
- 秒杀活动全流程管理（配置→预热→抢购→结果）
- 营销活动数据统计与分析

**不包含功能**：
- 订单支付流程（交易模块）
- 用户认证与权限（用户模块）
- 商品信息管理（商品模块）
- 消息推送（IM模块）

## 三、优惠券服务

### 3.1 功能概述

优惠券服务提供校园专属优惠券的全生命周期管理，支持满减券、直降券、品类券三种类型，贴合校园交易场景。

### 3.2 优惠券类型

| 类型 | 编码 | 说明 | 校园场景示例 |
|-----|------|------|------------|
| 满减券 | FULL_REDUCE | 满X元减Y元 | 满20减5（毕业季教材券） |
| 直降券 | DIRECT_REDUCE | 直接抵扣Y元 | 新人5元无门槛券 |
| 品类券 | CATEGORY | 指定品类可用 | 教材专享8折券 |

### 3.3 优惠券状态流转

```
┌─────────┐    领取     ┌─────────┐    使用    ┌─────────┐
│  可领取  │ ─────────▶ │  未使用  │ ─────────▶ │  已使用  │
└─────────┘            └────┬────┘            └─────────┘
                            │
                            │ 过期
                            ▼
                       ┌─────────┐
                       │  已过期  │
                       └─────────┘
```

### 3.4 核心功能设计

#### 3.4.1 优惠券创建（管理员）

**功能描述**：管理员创建新的优惠券，配置优惠券类型、金额、有效期等信息。

**业务规则**：
- 优惠券名称必填，建议贴合校园场景（如"毕业季满20减5券"）
- 满减券必须配置满减门槛金额
- 品类券必须关联有效的商品品类
- 有效期不能超过90天（贴合校园学期周期）
- 发放总数默认100张，最大不超过10000张

**接口设计**：
```
POST /coupon
Content-Type: application/json

Request:
{
  "couponName": "毕业季满20减5券",
  "couponType": "FULL_REDUCE",
  "fullAmount": 20.00,
  "reduceAmount": 5.00,
  "categoryId": null,
  "totalCount": 200,
  "startTime": "2024-06-01 00:00:00",
  "endTime": "2024-06-30 23:59:59",
  "enableStatus": 1
}

Response:
{
  "code": 200,
  "message": "创建成功",
  "data": null
}
```

#### 3.4.2 优惠券领取

**功能描述**：用户领取可用的优惠券。

**业务规则**：
- 用户必须完成校园认证才能领取
- 每个用户每张优惠券限领1张
- 优惠券库存不足时无法领取
- 优惠券已禁用或已过期时无法领取

**接口设计**：
```
POST /coupon/{couponId}/receive
Authorization: Bearer {token}

Response:
{
  "code": 200,
  "message": "领取成功",
  "data": null
}

Error Response:
{
  "code": 4002,
  "message": "优惠券已领完",
  "data": null
}
```

#### 3.4.3 优惠券核销

**功能描述**：订单支付时核销用户优惠券。

**业务规则**：
- 订单金额必须满足优惠券使用条件
- 品类券只能用于指定品类商品
- 优惠券必须在有效期内
- 每个订单只能使用一张优惠券
- 核销成功后更新优惠券状态为"已使用"

**核销流程**：
```
1. 校验优惠券状态（UNUSED）
2. 校验优惠券有效期
3. 校验订单金额是否满足条件
4. 校验品类限制（品类券）
5. 扣减优惠金额
6. 更新优惠券状态为USED
7. 关联订单ID
```

#### 3.4.4 优惠券过期处理

**功能描述**：定时任务处理过期优惠券。

**处理规则**：
- 每日凌晨1点执行
- 批量更新已过期优惠券状态为EXPIRED
- 发送过期提醒通知（可选）

## 四、秒杀服务

### 4.1 功能概述

秒杀服务提供限时抢购活动管理，适配校园用户规模，采用轻量化技术方案实现高并发抢购。

### 4.2 秒杀活动状态流转

```
┌─────────┐    开始时间    ┌─────────┐    结束时间    ┌─────────┐
│  未开始  │ ───────────▶ │  进行中  │ ───────────▶ │  已结束  │
│NOT_START│              │ON_GOING │              │   END   │
└─────────┘              └─────────┘              └─────────┘
```

### 4.3 技术方案

#### 4.3.1 整体架构

```
┌───────────────────────────────────────────────────────────────┐
│                         秒杀请求流程                           │
├───────────────────────────────────────────────────────────────┤
│                                                               │
│  用户请求 ──▶ Gateway限流 ──▶ Sentinel熔断 ──▶ 秒杀服务       │
│                                              │                │
│                                              ▼                │
│                                       Redis预扣库存           │
│                                              │                │
│                                              ▼                │
│                                       发送MQ消息              │
│                                              │                │
│                                              ▼                │
│                                       返回"请求已提交"        │
│                                                               │
│  ──────────────────────── 异步处理 ─────────────────────────  │
│                                                               │
│  MQ消费者 ──▶ 创建订单 ──▶ 更新DB库存 ──▶ 通知用户结果       │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

#### 4.3.2 库存管理方案

**方案说明**：采用Redis预扣库存 + DB最终一致性方案

**设计理由**：
- 校园用户规模≤10万，秒杀库存≤50件
- 无需分布式锁，Redis原子操作即可保证不超卖
- 简化架构，降低开发运维成本

**库存预热**：
```java
// 活动开始前30分钟预热库存到Redis
public void preloadStock(Long activityId) {
    SeckillActivity activity = activityMapper.selectById(activityId);
    String stockKey = "seckill:stock:" + activityId;
    redisTemplate.opsForValue().set(stockKey, activity.getTotalStock());
}
```

**库存扣减**：
```java
// Redis原子扣减
public boolean deductStock(Long activityId) {
    String stockKey = "seckill:stock:" + activityId;
    Long remain = redisTemplate.opsForValue().decrement(stockKey);
    return remain >= 0;
}
```

#### 4.3.3 限流方案

**Sentinel配置**：
- 秒杀接口QPS阈值：500（适配校园规模）
- 超出阈值返回"秒杀火爆，请稍后重试"
- 支持热点参数限流（按活动ID限流）

**网关层限流**：
- 基于IP限流：单IP每秒最多10次请求
- 基于用户限流：单用户每秒最多5次请求

#### 4.3.4 防刷机制

- 用户资格校验：仅校园认证用户可参与
- 单用户限购：每个活动每个用户限购1件
- 请求签名验证：防止接口被恶意调用
- 验证码机制：高并发场景触发验证码

### 4.4 核心功能设计

#### 4.4.1 秒杀活动配置（管理员）

- 活动时长不超过2小时
- 秒杀库存不超过商品实际库存
- 单次活动商品数量≤50件（校园规模）

## 五、秒杀提醒服务

**功能描述**：用户设置秒杀活动提醒。

**业务规则**：
- 活动必须为"未开始"状态
- 每个活动每个用户只能设置一个提醒

```
Authorization: Bearer {token}

{
  "code": 200,
  "message": "提醒设置成功",
  "data": null
```

#### 5.2.2 提醒推送

**推送规则**：
- 活动开始前5分钟触发
- 通过IM消息推送提醒
- 推送内容包含活动名称、开始时间

**推送内容**：
```
【秒杀提醒】您预约的"教材秒杀"活动即将在5分钟后开始，请做好准备！
```

## 六、数据统计服务

### 6.1 功能概述

数据统计服务提供营销活动效果分析，帮助运营人员优化营销策略。

### 6.2 统计指标

#### 6.2.1 优惠券统计

| 指标名称 | 计算公式 | 说明 |
|---------|---------|------|
| 领取率 | 已领数量/发放总数×100% | 优惠券吸引力指标 |
| 使用率 | 已用数量/已领数量×100% | 优惠券转化效果 |
| 核销金额 | SUM(抵扣金额) | 优惠成本统计 |
| ROI | 核销订单金额/核销金额 | 投入产出比 |

#### 6.2.2 秒杀统计

| 指标名称 | 计算公式 | 说明 |
|---------|---------|------|
| 参与人数 | COUNT(DISTINCT user_id) | 活动热度指标 |
| 成交率 | 成交数量/参与人数×100% | 活动转化效果 |
| 售罄时间 | 首次下单时间~库存清零时间 | 活动火爆程度 |
| 客单价 | 成交金额/成交订单数 | 用户消费能力 |

| 4003 | 优惠券已过期 | 提示用户领取其他优惠券 |
| 4004 | 优惠券不满足使用条件 | 提示用户凑单或更换优惠券 |

| 4006 | 优惠券已禁用 | 提示用户领取其他优惠券 |
| 5001 | 秒杀活动不存在 | 检查活动ID是否正确 |
| 5002 | 秒杀活动未开始 | 提示用户等待活动开始 |
| 5003 | 秒杀活动已结束 | 提示用户关注后续活动 |
| 5004 | 秒杀库存不足 | 提示用户下次早点参与 |
| 5005 | 用户已参与过该秒杀 | 每人限购一件 |

## 九、定时任务

### 9.1 任务清单

| 任务名称 | 执行周期 | 功能说明 |
|---------|---------|---------|
| 优惠券过期处理 | 每日01:00 | 批量更新过期优惠券状态 |
| 秒杀活动状态更新 | 每分钟 | 更新活动状态（未开始→进行中→已结束） |
| 秒杀库存预热 | 活动开始前30分钟 | 将库存加载到Redis |
| 秒杀提醒推送 | 活动开始前5分钟 | 推送秒杀提醒消息 |
| 数据统计汇总 | 每日02:00 | 汇总前一日营销数据 |

### 9.2 任务实现

```java
@Component
public class MarketingScheduledTasks {

    /**
     * 优惠券过期处理 - 每日01:00执行
     */
    @Scheduled(cron = "0 0 1 * * ?")
    public void processExpiredCoupons() {
        // 批量更新过期优惠券
        userCouponMapper.updateExpiredCoupons();
    }

    /**
     * 秒杀活动状态更新 - 每分钟执行
     */
    @Scheduled(cron = "0 * * * * ?")
    public void updateActivityStatus() {
        // 更新未开始→进行中
        activityMapper.updateToOngoing();
        // 更新进行中→已结束
        activityMapper.updateToEnded();
    }

    /**
     * 秒杀库存预热 - 活动开始前30分钟
     */
    @Scheduled(cron = "0 */5 * * * ?")
    public void preloadSeckillStock() {
        // 查询30分钟内开始的活动
        List<SeckillActivity> activities = activityMapper.findStartingSoon();
        activities.forEach(activity -> {
            String stockKey = "seckill:stock:" + activity.getId();
            redisTemplate.opsForValue().set(stockKey, activity.getTotalStock());
        });
    }

    /**
     * 秒杀提醒推送 - 活动开始前5分钟
     */
    @Scheduled(cron = "0 */5 * * * ?")
    public void pushSeckillReminders() {
        // 查询5分钟内开始的活动
        List<SeckillActivity> activities = activityMapper.findStartingIn5Minutes();
        activities.forEach(activity -> {
            // 查询设置提醒的用户
            List<Long> userIds = reminderMapper.findUserIdsByActivity(activity.getId());
            // 推送提醒消息
            imService.sendReminders(userIds, activity);
        });
    }
}
```

## 十、性能优化

### 10.1 缓存策略

| 缓存项 | Key格式 | 过期时间 | 说明 |
|-------|---------|---------|------|
| 优惠券详情 | coupon:info:{id} | 1小时 | 优惠券基础信息 |
| 可领取优惠券列表 | coupon:available | 5分钟 | 首页展示 |
| 秒杀活动详情 | seckill:info:{id} | 活动期间 | 活动基础信息 |
| 秒杀库存 | seckill:stock:{id} | 活动期间 | Redis预扣库存 |
| 今日秒杀列表 | seckill:today | 1分钟 | 首页展示 |

### 10.2 数据库优化

- 优惠券表：`coupon_name`、`coupon_type`、`enable_status` 字段加索引
- 用户优惠券表：`user_id`、`coupon_id`、`coupon_status` 字段加索引
- 秒杀活动表：`product_id`、`activity_status`、`start_time` 字段加索引

### 10.3 异步处理

- 秒杀下单：通过RabbitMQ异步处理，削峰填谷
- 提醒推送：通过RabbitMQ异步推送消息
- 统计汇总：通过RabbitMQ异步更新统计数据

## 十一、安全设计

### 11.1 接口安全

- 敏感接口（创建/删除）需管理员权限
- 用户操作接口需登录认证
- 秒杀接口需校园认证

### 11.2 防刷机制

- 限流：Sentinel接口限流
- 验证码：高并发场景触发验证码
- 请求签名：防止接口被恶意调用
- 用户限购：单用户限购一件

### 11.3 数据安全

- 优惠券发放记录可追溯
- 秒杀订单关联活动ID
- 操作日志记录关键操作
