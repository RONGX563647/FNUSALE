## 一、模块概述

### 1.1 模块定位

商品模块是校园二手交易平台的核心业务模块，负责商品的全生命周期管理，包括商品发布、品类管理、商品检索、状态管理、用户行为（收藏/点赞）等功能。

### 1.2 核心目标

- **简化发布**：极简表单 + AI拍照分类，发布流程≤3步
- **精准检索**：关键词检索 + 多维度筛选 + 距离排序
- **状态管理**：上架/下架/已成交/违规状态流转
- **用户行为**：收藏/点赞功能，支撑个性化推荐

### 1.3 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Spring Boot | 3.2+ | 应用框架 |
| MyBatis-Plus | 3.5.x | ORM框架 |
| Elasticsearch | 8.0 | 商品检索 |
| Redis | 7.0 | 缓存/热点数据 |
| MinIO/OSS | - | 图片存储 |
| 阿里云视觉AI | - | 拍照分类 |

**商品状态枚举（ProductStatus）**：

| 枚举值 | 说明 | 触发条件 |
|-------|------|---------|

| ON_SHELF | 上架 | 用户上架商品 |
| SOLD_OUT | 已成交 | 订单完成 |
| OFF_SHELF | 下架 | 用户下架/管理员下架 |
| ILLEGAL | 违规 | 管理员审核驳回 |

**新旧程度枚举（NewDegree）**：

| 枚举值 | 说明 | 描述 |
|-------|------|------|
| NEW | 全新 | 未拆封/未使用 |
| 90_NEW | 9成新 | 轻微使用痕迹 |
| 80_NEW | 8成新 | 明显使用痕迹，功能正常 |
| 70_NEW | 7成新 | 使用痕迹明显，有轻微瑕疵 |
| OLD | 老旧 | 功能正常，外观磨损严重 |

## 六、技术实现细节

### 6.1 缓存策略

#### 6.1.1 缓存Key设计

| Key格式 | 说明 | 过期时间 |
|---------|------|---------|
| `product:detail:{id}` | 商品详情 | 30分钟 |
| `product:list:page:{hash}` | 商品列表页 | 5分钟 |
| `product:category:tree` | 品类树 | 1小时 |
| `product:hot:list` | 热门商品 | 10分钟 |
| `user:behavior:{userId}:{productId}` | 用户行为 | 永久 |

#### 6.1.2 缓存更新策略

- **商品详情**：商品更新时删除缓存
- **商品列表**：定时刷新/商品变更时删除
