## 一、模块概述

### 1.1 模块定位

交易模块（fnusale-trade）是校园二手交易平台的核心业务模块，负责处理平台上所有交易相关的业务流程，包括订单管理、支付对接、线下自提确认、订单评价以及交易纠纷处理。

### 1.2 核心职责

| 职责 | 说明 |
|------|------|
| 订单管理 | 订单创建、查询、状态流转、取消、确认收货等 |
| 支付对接 | 对接微信/支付宝/校园卡支付，处理支付回调、退款 |
| 自提确认 | 校园线下自提场景的确认收货流程 |
| 订单评价 | 买家评价、卖家回复、评价统计 |
| 纠纷处理 | 交易纠纷申请、举证、管理员介入处理 |

### 1.3 设计原则

1. **流程简化**：仅保留「待付款→待自提→已成交」核心状态，无物流环节
2. **校园适配**：支持校园自提点管理、校园卡支付
3. **安全合规**：对接第三方支付，支持退款机制
4. **轻量高效**：适配校园用户规模（≤10万用户）

## 二、业务流程设计

### 2.1 订单状态流转

```
┌─────────────┐
│   创建订单   │
└──────┬──────┘
       │
       ▼
┌─────────────┐    24小时未付款    ┌─────────────┐
│   待付款    │ ──────────────────▶│   已取消    │
│   UNPAID   │                    │   CANCEL    │
└──────┬──────┘                    └─────────────┘
       │ 支付成功
       ▼
┌─────────────┐    买家确认/7天自动    ┌─────────────┐
│   待自提    │ ─────────────────────▶│   已成交    │
│ WAIT_PICK  │                        │  SUCCESS   │
└──────┬──────┘                        └─────────────┘
       │
       │ 申请退款
       ▼
┌─────────────┐
│   已退款    │
│  REFUNDED   │
└─────────────┘
```

### 2.2 交易完整流程

```
买家浏览商品 → 发起IM沟通 → 创建订单 → 支付 → 卖家备货 → 线下自提 → 确认收货 → 评价
                                                    ↓
                                              发起纠纷（可选）
```

### 2.3 核心流程说明

#### 2.3.1 创建订单流程

1. 买家选择商品、自提点、优惠券
2. 系统校验商品状态（上架中）、库存
3. 计算实付金额（商品价格 - 优惠券抵扣）
4. 生成订单编号（格式：XS + 日期 + 序号，如 XS20240101001）
5. 订单状态初始化为「待付款」
6. 设置24小时自动取消定时任务

#### 2.3.2 支付流程

1. 买家选择支付方式（微信/支付宝/校园卡）
2. 调用第三方支付接口获取支付链接
3. 买家完成支付
4. 第三方支付回调通知
5. 更新订单状态为「待自提」
6. 通知卖家备货

#### 2.3.3 自提确认流程

1. 卖家标记商品已备好（可选）
2. 买家线下自提
3. 买家扫码或手动确认收货
4. 订单状态变更为「已成交」
5. 商品状态变更为「已成交」
6. 触发评价流程

## 七、核心技术实现

### 7.1 订单编号生成

```java
// 格式: XS + 年月日 + 6位序号
// 示例: XS20240101000001
public String generateOrderNo() {
    String dateStr = LocalDate.now().format(DateTimeFormatter.BASIC_ISO_DATE);
    String key = "order:no:" + dateStr;
    Long seq = redisTemplate.opsForValue().increment(key);
    redisTemplate.expire(key, 1, TimeUnit.DAYS);
    return "XS" + dateStr + String.format("%06d", seq);
}
```

### 7.2 订单创建幂等性保证

```java
// 基于Redis实现幂等性校验
public Long createOrder(OrderCreateDTO dto) {
    String idempotentKey = "order:create:" + dto.getProductId() + ":" + userId;
    Boolean success = redisTemplate.opsForValue()
        .setIfAbsent(idempotentKey, "1", 10, TimeUnit.SECONDS);
    if (!success) {
        throw new BusinessException("请勿重复下单");
    }
    // ... 创建订单逻辑
}
```

### 7.3 支付状态补偿机制

```java
@Scheduled(fixedRate = 60000) // 每分钟执行一次
public void checkUnpaidOrders() {
    // 查询创建超过5分钟仍为待付款的订单
    List<Order> orders = orderMapper.selectUnpaidOrders(5);
    for (Order order : orders) {
        // 主动查询支付状态
        PayStatus status = paymentService.queryPayStatus(order.getId());
        if (status == PayStatus.PAID) {
            // 补偿更新订单状态
            orderService.updateOrderPaid(order.getId());
        }
    }
}
```
