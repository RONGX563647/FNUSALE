# 营销模块消息队列使用代码细节学习文档

## 一、文档概述

本文档深入讲解营销模块中 RocketMQ 消息队列的使用细节，包括 Topic/Tag 设计、事件定义、生产者实现、消费者实现、幂等性保证、顺序消息、延迟消息、批量消息等核心内容。

## 二、RocketMQ 基础概念

### 2.1 Topic 与 Tag 设计原则

```
┌─────────────────────────────────────────────────────────────┐
│                    RocketMQ 消息模型                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Topic（主题）                                               │
│  ├── 第一级消息分类                                          │
│  ├── 逻辑概念，物理上对应多个队列                             │
│  └── 命名规范：业务领域-动作（如 user-register）              │
│                                                             │
│  Tag（标签）                                                 │
│  ├── 第二级消息分类                                          │
│  ├── 同一 Topic 下的细分场景                                 │
│  └── 命名规范：具体操作（如 init-points、welcome）           │
│                                                             │
│  消息路由：                                                  │
│  Topic:Tag → 精确匹配                                       │
│  Topic:*   → 订阅该 Topic 下所有 Tag                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 三、事件定义详解

### 3.1 事件基类设计原则

```
┌─────────────────────────────────────────────────────────────┐
│                    事件设计原则                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 必须包含 eventId（事件唯一ID）                            │
│     - 用于幂等性处理                                        │
│     - UUID 生成                                            │
│                                                             │
│  2. 必须包含业务主键                                         │
│     - userId、couponId、activityId 等                       │
│     - 用于业务关联                                          │
│                                                             │
│  3. 必须包含时间戳                                           │
│     - 用于追溯和排序                                        │
│                                                             │
│  4. 使用 Lombok 注解                                         │
│     - @Data：getter/setter                                  │
│     - @Builder：构建者模式                                  │
│     - @NoArgsConstructor：无参构造（反序列化需要）            │
│     - @AllArgsConstructor：全参构造                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**嵌套类设计说明**:

```
为什么使用嵌套类 UserCouponInfo？

1. 场景区分：
   - 单个提醒：使用顶层字段（userId、couponName...）
   - 批量提醒：使用 userCouponList 列表

2. 代码复用：
   - UserCouponInfo 封装了用户优惠券的核心信息
   - 避免在多处重复定义相同字段

3. 类型安全：
   - 编译期检查字段类型
   - IDE 自动补全支持

使用示例：

    .userId(1001L)
    .couponName("满100减20")
    .reduceAmount(new BigDecimal("20"))
    .expireTime(LocalDateTime.now().plusDays(3))
    .build();

// 批量提醒
List<UserCouponInfo> list = Arrays.asList(
    UserCouponInfo.builder().userId(1001L).couponName("满100减20").build(),
    UserCouponInfo.builder().userId(1002L).couponName("满200减50").build()
);
CouponExpireReminderEvent event = CouponExpireReminderEvent.builder()
    .userCouponList(list)
    .build();
```

### 4.2 MarketingEventPublisherImpl 实现详解

**文件位置**: [MarketingEventPublisherImpl.java](../backend/fnusale-marketing/src/main/java/com/fnusale/marketing/service/impl/MarketingEventPublisherImpl.java)

#### 4.2.1 类结构与依赖

```java
@Slf4j
@Service
@RequiredArgsConstructor
public class MarketingEventPublisherImpl implements MarketingEventPublisher {

    private final RocketMQTemplate rocketMQTemplate;

    /**
     * 顺序消息队列选择器
     * 根据哈希键选择队列，相同哈希键的消息发送到同一队列
     */
    private final MessageQueueSelector queueSelector = (mqs, msg, arg) -> {
        String hashKey = (String) arg;
        int index = Math.abs(hashKey.hashCode()) % mqs.size();
        return mqs.get(index);
    };
}
```

**MessageQueueSelector 详解**:

```
┌─────────────────────────────────────────────────────────────┐
│                    顺序消息队列选择器                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  参数说明：                                                  │
│  - mqs：Topic 下的所有队列列表                               │
│  - msg：待发送的消息                                        │
│  - arg：哈希键（如 userId）                                  │
│                                                             │
│  选择逻辑：                                                  │
│  queueIndex = Math.abs(hashKey.hashCode()) % queueCount    │
│                                                             │
│  示例：                                                     │
│  Topic 有 4 个队列：[Queue0, Queue1, Queue2, Queue3]        │
│                                                             │
│  userId = "1001"                                            │
│  hashCode("1001") = 1507424                                 │
│  index = 1507424 % 4 = 0                                    │
│  → 选择 Queue0                                              │
│                                                             │
│  userId = "1002"                                            │
│  hashCode("1002") = 1507425                                 │
│  index = 1507425 % 4 = 1                                    │
│  → 选择 Queue1                                              │
│                                                             │
│  保证：                                                     │
│  相同 hashKey 的消息总是发送到同一队列                       │
│  同一队列内的消息按发送顺序消费                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 4.2.2 异步发送详解

```java
@Override
public void publishSeckillOrderEvent(SeckillOrderEvent event) {
    // 构建目的地：topic:tag
    String destination = RocketMQConstants.SECKILL_ORDER_TOPIC + ":" + RocketMQConstants.SECKILL_ORDER_TAG_CREATE;
    
    // 异步发送
    rocketMQTemplate.asyncSend(destination, event, new SendCallback() {
        @Override
        public void onSuccess(SendResult sendResult) {
            log.info("秒杀订单消息发送成功, eventId: {}, msgId: {}", 
                    event.getEventId(), sendResult.getMsgId());
        }

        @Override
        public void onException(Throwable e) {
            log.error("秒杀订单消息发送失败, eventId: {}, userId: {}", 
                    event.getEventId(), event.getUserId(), e);
        }
    });
}
```

#### 4.2.3 顺序发送详解

```java
public void publishSeckillOrderEventOrdered(SeckillOrderEvent event, String hashKey) {
    String destination = RocketMQConstants.SECKILL_ORDER_TOPIC + ":" + RocketMQConstants.SECKILL_ORDER_TAG_CREATE;
```

**批量顺序发送实现**:

```java
@Override
public void publishCouponGrantBatchOrdered(List<CouponGrantEvent> events, List<String> hashKeys) {
    if (events == null || events.isEmpty()) {
        return;
    }

    if (hashKeys == null || hashKeys.size() != events.size()) {
        throw new IllegalArgumentException("hashKeys 数量必须与 events 数量一致");
    }

    String destination = RocketMQConstants.COUPON_GRANT_TOPIC + ":" + RocketMQConstants.COUPON_GRANT_TAG_BATCH;

    int successCount = 0;
    int failCount = 0;

    // 逐条发送顺序消息
    for (int i = 0; i < events.size(); i++) {
        CouponGrantEvent event = events.get(i);
        String hashKey = hashKeys.get(i);

        try {
            SendResult sendResult = rocketMQTemplate.syncSendOrderly(destination, event, hashKey);
            successCount++;
            log.debug("批量顺序消息发送成功, eventId: {}, hashKey: {}, msgId: {}",
                    event.getEventId(), hashKey, sendResult.getMsgId());
        } catch (Exception e) {
            failCount++;
            log.error("批量顺序消息发送失败, eventId: {}, hashKey: {}", event.getEventId(), hashKey, e);
        }
    }

    log.info("批量优惠券发放顺序消息发送完成, 成功: {}, 失败: {}", successCount, failCount);
}
```

### 5.2 CouponGrantConsumer - 优惠券发放消费者

**文件位置**: [CouponGrantConsumer.java](../backend/fnusale-marketing/src/main/java/com/fnusale/marketing/mq/consumer/CouponGrantConsumer.java)

```java
@Slf4j
@Component
@RocketMQMessageListener(
        topic = RocketMQConstants.COUPON_GRANT_TOPIC,
        consumerGroup = "coupon-grant-consumer-group",
        selectorExpression = RocketMQConstants.COUPON_GRANT_TAG_BATCH
)
@RequiredArgsConstructor
public class CouponGrantConsumer implements RocketMQListener<CouponGrantEvent> {

    private final UserCouponMapper userCouponMapper;
    private final CouponMapper couponMapper;
    private final StringRedisTemplate redisTemplate;

    private static final String COUPON_GRANT_KEY_PREFIX = "coupon:grant:";

    @Override
    public void onMessage(CouponGrantEvent event) {
        Long couponId = event.getCouponId();
        Long userId = event.getUserId();
        String eventId = event.getEventId();

        log.info("收到优惠券发放消息, couponId: {}, userId: {}, eventId: {}", 
                couponId, userId, eventId);

        // ========== 1. 幂等性检查 ==========
        String idempotentKey = COUPON_GRANT_KEY_PREFIX + eventId;
        Boolean success = redisTemplate.opsForValue().setIfAbsent(
                idempotentKey, "1", 24, TimeUnit.HOURS);
        if (Boolean.FALSE.equals(success)) {
            log.info("优惠券已发放，跳过, eventId: {}", eventId);
            return;
        }

        try {
            // ========== 2. 业务校验 ==========
            // 检查优惠券是否存在
            if (couponMapper.selectById(couponId) == null) {
                log.warn("优惠券不存在, couponId: {}", couponId);
                return;
            }

            // 检查用户是否已领取
            if (userCouponMapper.countByUserAndCoupon(userId, couponId) > 0) {
                log.info("用户已领取该优惠券, userId: {}, couponId: {}", userId, couponId);
                return;
            }

            // ========== 3. 扣减库存（乐观锁）==========
            int rows = couponMapper.incrementReceivedCount(couponId);
            if (rows == 0) {
                log.warn("优惠券库存不足, couponId: {}", couponId);
                return;
            }

            // ========== 4. 创建用户优惠券记录 ==========
            UserCoupon userCoupon = UserCoupon.builder()
                    .userId(userId)
                    .couponId(couponId)
                    .receiveTime(LocalDateTime.now())
                    .expireTime(event.getExpireTime())
                    .couponStatus(MarketingConstants.COUPON_STATUS_UNUSED)
                    .createTime(LocalDateTime.now())
                    .build();

            userCouponMapper.insert(userCoupon);

            log.info("优惠券发放成功, couponId: {}, userId: {}", couponId, userId);
        } catch (Exception e) {
            // ========== 5. 异常处理 ==========
            // 删除幂等性 Key，允许重试
            redisTemplate.delete(idempotentKey);
            log.error("优惠券发放失败, couponId: {}, userId: {}", couponId, userId, e);
            throw e;  // 抛出异常触发 MQ 重试
        }
    }
}
```

```java
@Component
@RocketMQMessageListener(
        topic = RocketMQConstants.COUPON_RECEIVE_TOPIC,
        consumerGroup = "coupon-receive-consumer-group",
        selectorExpression = RocketMQConstants.COUPON_RECEIVE_TAG_RECEIVE
)
@RequiredArgsConstructor
public class CouponReceiveConsumer implements RocketMQListener<CouponReceiveEvent> {

    private final UserCouponMapper userCouponMapper;
    private final CouponMapper couponMapper;
    private final StringRedisTemplate redisTemplate;

    private static final String COUPON_RECEIVE_KEY_PREFIX = "coupon:receive:";

    @Override
    public void onMessage(CouponReceiveEvent event) {
        Long couponId = event.getCouponId();
        Long userId = event.getUserId();
        String eventId = event.getEventId();

        log.info("收到优惠券领取消息, couponId: {}, userId: {}, eventId: {}", 
                couponId, userId, eventId);

        // 幂等性检查
        String idempotentKey = COUPON_RECEIVE_KEY_PREFIX + eventId;
        Boolean success = redisTemplate.opsForValue().setIfAbsent(
                idempotentKey, "1", 24, TimeUnit.HOURS);
        if (Boolean.FALSE.equals(success)) {
            log.info("优惠券已领取，跳过, eventId: {}", eventId);
            return;
        }

        try {
            // 再次检查是否已领取（防止重复入库）
            if (userCouponMapper.countByUserAndCoupon(userId, couponId) > 0) {
                log.info("用户已领取该优惠券, userId: {}, couponId: {}", userId, couponId);
                return;
            }

            // 创建用户优惠券记录
            UserCoupon userCoupon = UserCoupon.builder()
                    .userId(userId)
                    .couponId(couponId)
                    .receiveTime(event.getReceiveTime())
                    .expireTime(event.getExpireTime())
                    .couponStatus(MarketingConstants.COUPON_STATUS_UNUSED)
                    .createTime(LocalDateTime.now())
                    .build();

            userCouponMapper.insert(userCoupon);

            log.info("优惠券领取入库成功, couponId: {}, userId: {}, userCouponId: {}",
                    couponId, userId, userCoupon.getId());
        } catch (Exception e) {
            redisTemplate.delete(idempotentKey);
            log.error("优惠券领取入库失败, couponId: {}, userId: {}", couponId, userId, e);
            throw e;
        }
    }
}
```

### 5.5 CouponExpireReminderConsumer - 优惠券过期提醒消费者

**文件位置**: [CouponExpireReminderConsumer.java](../backend/fnusale-marketing/src/main/java/com/fnusale/marketing/mq/consumer/CouponExpireReminderConsumer.java)

```java
@Slf4j
@Component
@RocketMQMessageListener(
        topic = RocketMQConstants.COUPON_EXPIRE_REMINDER_TOPIC,
        consumerGroup = "coupon-expire-reminder-consumer-group",
        selectorExpression = RocketMQConstants.COUPON_EXPIRE_REMINDER_TAG_NOTIFY
)
@RequiredArgsConstructor
public class CouponExpireReminderConsumer implements RocketMQListener<CouponExpireReminderEvent> {

    private final StringRedisTemplate redisTemplate;

    private static final String COUPON_EXPIRE_REMINDER_KEY_PREFIX = "coupon:expire:reminder:";
    private static final DateTimeFormatter DATE_FORMATTER = DateTimeFormatter.ofPattern("MM月dd日");

    @Override
    public void onMessage(CouponExpireReminderEvent event) {
        String eventId = event.getEventId();

        log.info("收到优惠券过期提醒消息, eventId: {}", eventId);

        // 幂等性检查
        String idempotentKey = COUPON_EXPIRE_REMINDER_KEY_PREFIX + eventId;
        Boolean success = redisTemplate.opsForValue().setIfAbsent(
                idempotentKey, "1", 24, TimeUnit.HOURS);
        if (Boolean.FALSE.equals(success)) {
            log.info("优惠券过期提醒已处理，跳过, eventId: {}", eventId);
            return;
        }

        try {
            // 处理批量提醒
            if (event.getUserCouponList() != null && !event.getUserCouponList().isEmpty()) {
                processBatchReminder(event);
            } else {
                // 单个提醒
                processSingleReminder(event);
            }

            log.info("优惠券过期提醒发送成功, eventId: {}", eventId);
        } catch (Exception e) {
            redisTemplate.delete(idempotentKey);
            log.error("优惠券过期提醒发送失败, eventId: {}", eventId, e);
            throw e;
        }
    }

    /**
     * 处理批量提醒
     */
    private void processBatchReminder(CouponExpireReminderEvent event) {
        for (CouponExpireReminderEvent.UserCouponInfo info : event.getUserCouponList()) {
            try {
                sendExpireNotification(info.getUserId(), info.getCouponName(),
                        info.getReduceAmount(), info.getExpireTime());
            } catch (Exception e) {
                log.warn("发送优惠券过期提醒失败, userId: {}, userCouponId: {}",
                        info.getUserId(), info.getUserCouponId(), e);
            }
        }
    }

    /**
     * 处理单个提醒
     */
    private void processSingleReminder(CouponExpireReminderEvent event) {
        sendExpireNotification(event.getUserId(), event.getCouponName(),
                event.getReduceAmount(), event.getExpireTime());
    }

    /**
     * 发送过期提醒通知
     */
    private void sendExpireNotification(Long userId, String couponName,
                                         java.math.BigDecimal reduceAmount,
                                         java.time.LocalDateTime expireTime) {
        String expireDateStr = expireTime.format(DATE_FORMATTER);
        String title = "优惠券即将过期";
        String content = String.format("您的优惠券「%s」（优惠%.2f元）将于%s过期，请尽快使用！",
                couponName, reduceAmount, expireDateStr);

        // TODO: 调用 IM 服务发送站内消息
        // imClient.sendSystemMessage(userId, title, content, "COUPON_EXPIRE");

        log.info("发送优惠券过期提醒: userId={}, title={}, content={}", userId, title, content);
    }
}
```

## 六、幂等性保证机制

### 6.1 幂等性概念

```
┌─────────────────────────────────────────────────────────────┐
│                    幂等性概念                                │

│                                                             │
```

### 6.2 Redis SETNX 实现幂等性

```java
/**
 * 幂等性检查
 * 使用 Redis SETNX 实现
 */
String idempotentKey = "coupon:grant:" + eventId;

// SETNX + EX 原子操作
// 如果 key 不存在，设置成功，返回 true
// 如果 key 已存在，设置失败，返回 false
Boolean success = redisTemplate.opsForValue().setIfAbsent(
        idempotentKey,    // key
        "1",              // value
        24,               // 过期时间
        TimeUnit.HOURS    // 时间单位
);

if (Boolean.FALSE.equals(success)) {
    // 已处理过，跳过
    log.info("消息已处理，跳过, eventId: {}", eventId);
    return;
}

try {
    // 执行业务逻辑
    doBusiness();
} catch (Exception e) {
    // 失败时删除幂等性 Key，允许重试
    redisTemplate.delete(idempotentKey);
    throw e;
}
```

```java
@Slf4j
@Component
@RocketMQMessageListener(
        topic = "your-topic",
        consumerGroup = "your-consumer-group",
        selectorExpression = "your-tag",
        maxReconsumeTimes = 3
)
@RequiredArgsConstructor
public class YourConsumer implements RocketMQListener<YourEvent> {

    private final StringRedisTemplate redisTemplate;
    private final YourService yourService;

    private static final String IDEMPOTENT_KEY_PREFIX = "your:business:";

    @Override
    public void onMessage(YourEvent event) {
        String eventId = event.getEventId();
        log.info("收到消息, eventId: {}", eventId);

        // 1. 幂等性检查
        String idempotentKey = IDEMPOTENT_KEY_PREFIX + eventId;
        Boolean success = redisTemplate.opsForValue().setIfAbsent(
                idempotentKey, "1", 24, TimeUnit.HOURS);
        if (Boolean.FALSE.equals(success)) {
            log.info("消息已处理，跳过, eventId: {}", eventId);
            return;
        }

        try {
            // 2. 业务校验
            if (!validate(event)) {
                log.warn("业务校验失败, eventId: {}", eventId);
                return;  // 业务校验失败，不重试
            }

            // 3. 执行业务逻辑
            yourService.doBusiness(event);

            log.info("消息处理成功, eventId: {}", eventId);
        } catch (BusinessException e) {
            // 4. 业务异常：记录日志，不重试
            log.warn("业务异常, eventId: {}, error: {}", eventId, e.getMessage());
        } catch (Exception e) {
            // 5. 系统异常：删除幂等性 Key，触发重试
            redisTemplate.delete(idempotentKey);
            log.error("系统异常, eventId: {}", eventId, e);
            throw e;
        }
    }

    private boolean validate(YourEvent event) {
        // 业务校验逻辑
        return true;
    }
}
```

### 12.1 RocketMQDLQConfig - 死信队列配置

**文件位置**: [RocketMQDLQConfig.java](../backend/fnusale-marketing/src/main/java/com/fnusale/marketing/config/RocketMQDLQConfig.java)

```java
/**
 * RocketMQ 死信队列配置
 *
 * 死信队列（Dead Letter Queue）用于存储无法正常消费的消息。
 * 当消息消费失败并超过最大重试次数后，会被转移到死信队列。
 *
 * 死信队列命名规则：
 * - Topic: %DLQ% + 消费者组名
 * - 示例: %DLQ%coupon-grant-consumer-group
 *
 * 处理策略：
 * 1. 记录死信消息到持久化存储（Redis/数据库）
 * 2. 发送告警通知（邮件/短信/钉钉）
 * 3. 人工介入处理
 * 4. 定期清理已处理的死信消息
 */
@Slf4j
@Configuration
public class RocketMQDLQConfig {

    /**
     * 死信队列 Topic 前缀
     */
    public static final String DLQ_TOPIC_PREFIX = "%DLQ%";

    /**
     * 死信消息保留天数
     */
    private static final int DLQ_MESSAGE_RETENTION_DAYS = 7;

    /**
     * 死信消息处理 Key 前缀
     */
    public static final String DLQ_PROCESSED_KEY_PREFIX = "dlq:processed:";

    /**
     * 判断是否为死信队列 Topic
     */
    public static boolean isDLQTopic(String topic) {
        return topic != null && topic.startsWith(DLQ_TOPIC_PREFIX);
    }

    /**
     * 从死信队列 Topic 中提取消费者组名
     */
    public static String extractConsumerGroup(String dlqTopic) {
        if (!isDLQTopic(dlqTopic)) {
            return null;
        }
        return dlqTopic.substring(DLQ_TOPIC_PREFIX.length());
    }

    /**
     * 构建死信队列 Topic
     */
    public static String buildDLQTopic(String consumerGroup) {
        return DLQ_TOPIC_PREFIX + consumerGroup;
    }
}
```

### 12.2 RocketMQMetricsConfig - 消息监控配置

**文件位置**: [RocketMQMetricsConfig.java](../backend/fnusale-marketing/src/main/java/com/fnusale/marketing/config/RocketMQMetricsConfig.java)

```java
/**
 * RocketMQ 消息监控配置
 *
 * 集成 Micrometer 实现消息发送和消费的监控指标采集。
 * 支持与 Prometheus、Grafana 等监控系统集成。
 *
 * 监控指标：
 * - 消息发送数量（rocketmq_producer_send_total）
 * - 消息发送耗时（rocketmq_producer_send_duration）
 * - 消息消费数量（rocketmq_consumer_consume_total）
 * - 消息消费耗时（rocketmq_consumer_consume_duration）
 * - 消费失败数量（rocketmq_consumer_consume_failed）
 */
@Slf4j
@Configuration
@RequiredArgsConstructor
public class RocketMQMetricsConfig {

    private final MeterRegistry meterRegistry;

    /**
     * 消息发送计数器缓存
     */
    private final Map<String, Counter> sendCounters = new ConcurrentHashMap<>();

    /**
     * 消息消费计数器缓存
     */
    private final Map<String, Counter> consumeCounters = new ConcurrentHashMap<>();

    /**
     * 记录消息发送成功
     */
    public void recordSendSuccess(String topic) {
        getSendCounter(topic, "success").increment();
    }

    /**
     * 记录消息发送失败
     */
    public void recordSendFailure(String topic) {
        getSendCounter(topic, "failure").increment();
    }

    /**
     * 记录消息消费成功
     */
    public void recordConsumeSuccess(String topic, String consumerGroup) {
        getConsumeCounter(topic, consumerGroup, "success").increment();
    }

    /**
     * 记录消息消费失败
     */
    public void recordConsumeFailure(String topic, String consumerGroup) {
        getConsumeCounter(topic, consumerGroup, "failure").increment();
    }

    /**
     * 获取消息发送计数器
     */
    public Counter getSendCounter(String topic, String result) {
        String key = topic + ":" + result;
        return sendCounters.computeIfAbsent(key, k -> 
                Counter.builder("rocketmq_producer_send_total")
                        .description("Total messages sent")
                        .tag("topic", topic)
                        .tag("result", result)
                        .tag("service", "fnusale-marketing")
                        .register(meterRegistry)
        );
    }

    /**
     * 获取消息消费计数器
     */
    public Counter getConsumeCounter(String topic, String consumerGroup, String result) {
        String key = topic + ":" + consumerGroup + ":" + result;
        return consumeCounters.computeIfAbsent(key, k ->
                Counter.builder("rocketmq_consumer_consume_total")
                        .description("Total messages consumed")
                        .tag("topic", topic)
                        .tag("consumer_group", consumerGroup)
                        .tag("result", result)
                        .tag("service", "fnusale-marketing")
                        .register(meterRegistry)
        );
    }

    /**
     * 获取消息发送计时器
     */
    public Timer getSendTimer(String topic) {
        return Timer.builder("rocketmq_producer_send_duration")
                .description("Message send duration")
                .tag("topic", topic)
                .tag("service", "fnusale-marketing")
                .register(meterRegistry);
    }

    /**
     * 获取消息消费计时器
     */
    public Timer getConsumeTimer(String topic, String consumerGroup) {
        return Timer.builder("rocketmq_consumer_consume_duration")
                .description("Message consume duration")
                .tag("topic", topic)
                .tag("consumer_group", consumerGroup)
                .tag("service", "fnusale-marketing")
                .register(meterRegistry);
    }
}
```

### 12.3 RocketMQTraceConfig - 消息轨迹配置

**文件位置**: [RocketMQTraceConfig.java](../backend/fnusale-marketing/src/main/java/com/fnusale/marketing/config/RocketMQTraceConfig.java)

```java
/**
 * RocketMQ 消息轨迹配置
 *
 * 消息轨迹功能可以记录消息从生产到消费的全过程，便于问题排查和性能分析。
 *
 * 轨迹信息包括:
 * - 生产时间、Broker 地址、消息 ID
 * - 发送耗时、发送状态
 * - 消费时间、消费耗时、消费状态
 *
 * 开启方式:
 * 1. Broker 端配置 traceTopicEnable=true
 * 2. 生产者端通过 rocketmq.producer.enable-msg-trace=true 配置
 * 3. 消费者端通过 @RocketMQMessageListener 的属性配置
 *
 * 查询轨迹:
 * 1. RocketMQ Console 的消息轨迹查询功能
 * 2. 通过代码查询 TRACE_TOPIC
 */
@Slf4j
@Configuration
public class RocketMQTraceConfig {

    @Value("${rocketmq.name-server:localhost:9876}")
    private String nameServer;

    /**
     * 轨迹 Topic
     * 默认为 RMQ_SYS_TRACE_TOPIC
     */
    public static final String TRACE_TOPIC = "RMQ_SYS_TRACE_TOPIC";

    /**
     * RocketMQ Console 地址（用于构建轨迹查询链接）
     */
    @Value("${rocketmq.console.url:http://localhost:8080}")
    private String consoleUrl;

    /**
     * 构建轨迹查询 URL
     */
    public String buildTraceQueryUrl(String messageId) {
        return consoleUrl + "/#/messageTrace?messageId=" + messageId;
    }

    /**
     * 记录消息轨迹日志
     */
    public void logTrace(String topic, String messageId, String operation, 
                         String status, long durationMs) {
        log.info("消息轨迹: topic={}, messageId={}, operation={}, status={}, durationMs={}, traceUrl={}",
                topic, messageId, operation, status, durationMs, buildTraceQueryUrl(messageId));
    }
}
```
