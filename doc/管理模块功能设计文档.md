## 一、模块概述

### 1.1 模块定位

管理模块（fnusale-admin）是FNUSALE校园二手交易平台的运营管理核心模块，承担着平台运营、审核、监控、配置的关键职责。作为校园专属二手交易平台，管理模块的核心价值在于：

- **运营效率**：提供可视化运营管理界面，降低运营人员操作门槛
- **内容安全**：审核违规商品，保障平台内容合规
- **用户管理**：管理用户认证、信誉体系，维护平台秩序
- **数据洞察**：提供运营数据统计，辅助决策优化

### 1.2 模块职责

| 职责领域 | 具体内容 |
|---------|---------|
| 商品审核 | 商品发布审核、违规处理、强制下架、审核记录管理 |
| 用户管理 | 用户认证审核、封禁管理、信誉分调整、用户查询 |
| 数据统计 | 运营数据概览、趋势分析、报表导出、活动效果分析 |
| 系统配置 | 系统参数配置、校园围栏配置、秒杀参数配置 |
| 纠纷处理 | 交易纠纷处理、投诉管理、仲裁记录 |

### 1.3 模块边界

**包含功能**：
- 商品审核与违规处理
- 用户认证审核与账户管理
- 运营数据统计与分析
- 系统参数配置管理
- 交易纠纷处理

**不包含功能**：
- 商品发布与展示（商品模块）
- 用户注册登录（用户模块）
- 订单交易流程（交易模块）
- 营销活动配置（营销模块）

## 三、商品审核服务

### 3.1 功能概述

商品审核服务提供商品发布审核、违规处理功能，保障平台内容合规，维护校园交易安全。

### 3.2 审核流程

```
┌─────────────┐    提交审核    ┌─────────────┐
│  用户发布   │ ─────────────▶ │  待审核     │
│   商品      │               │  PENDING    │
└─────────────┘               └──────┬──────┘
                                     │
                    ┌────────────────┼────────────────┐
                    │                │                │
                    ▼                ▼                ▼
              ┌──────────┐    ┌──────────┐    ┌──────────┐
              │ 审核通过  │    │ 审核驳回  │    │ AI辅助   │
              │  PASS    │    │  REJECT  │    │ 预审核   │
              └────┬─────┘    └────┬─────┘    └──────────┘
                   │               │
                   ▼               ▼
              ┌──────────┐    ┌──────────┐
              │  商品上架 │    │ 修改后   │
              │ ON_SHELF │    │ 重新提交  │
              └──────────┘    └──────────┘
```

### 3.3 核心功能设计

#### 3.3.1 获取待审核商品列表

**功能描述**：管理员获取待审核状态的商品列表，支持分页查询。

**业务规则**：
- 仅返回待审核状态的商品
- 按提交时间倒序排列
- 显示商品基本信息、发布者信息

**接口设计**：
```
GET /admin/audit/pending?pageNum=1&pageSize=10
Authorization: Bearer {admin_token}

Response:
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 100,
    "list": [
      {
        "productId": 1001,
        "productName": "大学物理教材",
        "price": 25.00,
        "categoryName": "教材",
        "publisherId": 2001,
        "publisherName": "张三",
        "publishTime": "2024-01-01 10:00:00",
        "mainImageUrl": "https://xxx.com/image.jpg",
        "productDesc": "九成新，无笔记痕迹"
      }
    ]
  }
}
```

#### 3.3.2 审核通过

**功能描述**：管理员审核通过商品，商品状态变更为上架。

**业务规则**：
- 商品状态变更为ON_SHELF
- 记录审核人和审核时间
- 发送审核通过通知给发布者

**接口设计**：
```
PUT /admin/audit/{productId}/pass
Authorization: Bearer {admin_token}

Response:
{
  "code": 200,
  "message": "审核通过",
  "data": null
}
```

#### 3.3.3 审核驳回

**功能描述**：管理员驳回商品，填写驳回原因。

**业务规则**：
- 商品状态变更为OFF_SHELF
- 必须填写驳回原因
- 记录审核人和驳回原因
- 发送审核驳回通知给发布者

**驳回原因预设值**：
| 原因类型 | 说明 |
|---------|------|
| 图片不清晰 | 商品图片模糊，无法辨别商品详情 |
| 信息不完整 | 商品描述不完整，缺少关键信息 |
| 价格异常 | 商品定价明显不合理 |
| 违规商品 | 商品属于违禁品或禁止交易品类 |
| 非校园商品 | 商品不适合在校园平台交易 |
| 其他 | 其他原因（需手动填写） |

**接口设计**：
```
PUT /admin/audit/{productId}/reject?reason=图片不清晰
Authorization: Bearer {admin_token}

Response:
{
  "code": 200,
  "message": "审核驳回",
  "data": null
}
```

#### 3.3.4 批量审核通过

**功能描述**：管理员批量审核通过多个商品，提升审核效率。

**业务规则**：
- 一次最多批量审核50个商品
- 仅支持待审核状态的商品
- 事务处理，保证数据一致性

**接口设计**：
```
PUT /admin/audit/batch/pass
Authorization: Bearer {admin_token}
Content-Type: application/json

Request:
{
  "productIds": [1001, 1002, 1003]
}

Response:
{
  "code": 200,
  "message": "批量审核完成",
  "data": {
    "successCount": 3,
    "failCount": 0
  }
}
```

#### 3.3.5 强制下架

**功能描述**：管理员强制下架已上架的违规商品。

**业务规则**：
- 商品状态变更为ILLEGAL
- 必须填写下架原因
- 记录操作人和下架原因
- 发送违规下架通知给发布者
- 扣减发布者信誉分（可选）

**接口设计**：
```
PUT /admin/audit/{productId}/force-off?reason=违规商品
Authorization: Bearer {admin_token}

Response:
{
  "code": 200,
  "message": "已强制下架",
  "data": null
}
```

### 3.4 AI辅助审核

#### 3.4.1 功能说明

集成AI能力，辅助管理员进行商品审核，提升审核效率。

|-----|------|-------|
| 图片审核 | 识别违规图片、敏感内容 | 95%+ |
| 文本审核 | 识别敏感词、违规描述 | 90%+ |
| 品类校验 | 校验商品品类与图片是否匹配 | 85%+ |

#### 3.4.2 审核建议

```java
/**

public class AiAuditSuggestionVO {
```

## 四、用户管理服务

用户管理服务提供用户认证审核、账户管理、信誉分管理功能，维护平台用户秩序。

```
       │
```

### 4.3 核心功能设计

#### 4.3.1 用户列表查询

**功能描述**：管理员分页查询用户列表，支持多条件筛选。

**业务规则**：
- 支持按用户名、认证状态、身份类型筛选
- 默认按注册时间倒序排列
- 敏感信息脱敏显示

**接口设计**：
```
GET /admin/user/page?username=张三&authStatus=AUTH_SUCCESS&identityType=STUDENT&pageNum=1&pageSize=10
Authorization: Bearer {admin_token}

Response:
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 1000,
    "list": [
      {
        "userId": 2001,
        "username": "张三",
        "phone": "138****8000",
        "studentTeacherId": "****1234",
        "identityType": "STUDENT",
        "authStatus": "AUTH_SUCCESS",
        "creditScore": 95,
        "status": "NORMAL",
        "registerTime": "2024-01-01 10:00:00"
      }
    ]
  }
}
```

#### 4.3.2 认证审核

**功能描述**：管理员审核用户提交的校园认证申请。

**认证状态流转**：
```
UNAUTH ──提交认证──▶ UNDER_REVIEW ──审核通过──▶ AUTH_SUCCESS
                              │
                              └──审核驳回──▶ AUTH_FAILED
                                    │
                                    └──重新提交──▶ UNDER_REVIEW
```

**审核通过接口**：
```
PUT /admin/user/auth/{userId}/pass
Authorization: Bearer {admin_token}

Response:
{
  "code": 200,
  "message": "认证通过",
  "data": null
}
```

**审核驳回接口**：
```
PUT /admin/user/auth/{userId}/reject?reason=校园卡信息模糊
Authorization: Bearer {admin_token}

Response:
{
  "code": 200,
  "message": "认证驳回",
  "data": null
}
```

#### 4.3.3 用户封禁管理

**功能描述**：管理员封禁/解封违规用户。

**封禁规则**：
- 封禁用户无法登录、发布、交易
- 必须填写封禁原因
- 记录封禁操作日志

**封禁接口**：
```
PUT /admin/user/{userId}/ban?reason=发布违规商品
Authorization: Bearer {admin_token}

Response:
{
  "code": 200,
  "message": "封禁成功",
  "data": null
}
```

**解封接口**：
```
PUT /admin/user/{userId}/unban
Authorization: Bearer {admin_token}

Response:
{
  "code": 200,
  "message": "解封成功",
  "data": null
}
```

#### 4.3.4 信誉分调整

**功能描述**：管理员手动调整用户信誉分。

**业务规则**：
- 支持加分和减分
- 必须填写调整原因
- 信誉分范围：0-100
- 信誉分<60限制发布和参与秒杀

**接口设计**：
```
PUT /admin/user/{userId}/credit?score=-10&reason=交易纠纷责任方
Authorization: Bearer {admin_token}

Response:
{
  "code": 200,
  "message": "信誉分调整成功",
  "data": {
    "beforeScore": 95,
    "afterScore": 85
  }
}
```

### 4.4 信誉分规则

#### 4.4.1 加分规则

| 行为 | 分值 | 说明 |
|-----|------|------|
| 交易成功 | +1 | 每次成功交易 |
| 获得好评 | +2 | 收到好评评价 |
| 连续交易无纠纷 | +5 | 连续10次交易无纠纷 |
| 举报违规属实 | +3 | 举报违规商品经核实 |

#### 4.4.2 减分规则

| 行为 | 分值 | 说明 |
|-----|------|------|
| 发布违规商品 | -10 | 商品被强制下架 |
| 交易纠纷责任方 | -5 | 纠纷处理判定责任方 |
| 恶意取消订单 | -3 | 下单后无理由取消 |
| 收到差评 | -2 | 收到差评评价 |
| 账号封禁 | -20 | 账号被封禁 |

## 五、数据统计服务

### 5.1 功能概述

数据统计服务提供平台运营数据统计分析，帮助运营人员了解平台运营状况，优化运营策略。

### 5.2 统计指标体系

#### 5.2.1 核心运营指标

| 指标分类 | 指标名称 | 计算公式 | 说明 |
|---------|---------|---------|------|
| 用户指标 | 新增用户数 | COUNT(当日注册用户) | 用户增长情况 |
| 用户指标 | 活跃用户数 | COUNT(当日登录用户) | 用户活跃度 |
| 用户指标 | 认证通过率 | 认证通过数/申请数×100% | 认证转化效率 |
| 商品指标 | 商品发布数 | COUNT(当日发布商品) | 商品供给情况 |
| 商品指标 | 商品审核通过率 | 审核通过数/审核总数×100% | 商品质量情况 |
| 交易指标 | 成交订单数 | COUNT(当日成交订单) | 交易活跃度 |
| 交易指标 | 成交金额 | SUM(当日成交金额) | 交易规模 |
| 交易指标 | 成交转化率 | 成交订单数/下单数×100% | 交易效率 |

#### 5.2.2 营销活动指标

| 指标分类 | 指标名称 | 计算公式 | 说明 |
|---------|---------|---------|------|
| 优惠券 | 领取率 | 已领数量/发放总数×100% | 优惠券吸引力 |
| 优惠券 | 使用率 | 已用数量/已领数量×100% | 优惠券转化效果 |
| 秒杀 | 参与人数 | COUNT(DISTINCT 参与用户) | 活动热度 |
| 秒杀 | 售罄率 | 售出数量/总库存×100% | 活动效果 |

#### 5.2.3 AI能力指标

| 指标名称 | 计算公式 | 说明 |
|---------|---------|------|
| AI分类准确率 | 正确分类数/总分类数×100% | AI分类能力评估 |
| AI审核准确率 | AI判断正确数/总判断数×100% | AI审核效果评估 |

### 11.1 错误码定义

| 错误码 | 说明 | 处理建议 |
|-------|------|---------|
| 6001 | 商品不存在 | 检查商品ID是否正确 |
| 6002 | 商品状态不允许审核 | 检查商品当前状态 |
| 6003 | 用户不存在 | 检查用户ID是否正确 |
| 6004 | 用户已封禁 | 提示用户已被封禁 |
| 6005 | 用户已认证 | 无需重复认证 |
| 6006 | 配置项不存在 | 检查配置键是否正确 |
| 6007 | 配置值格式错误 | 检查配置值格式 |
| 6008 | 权限不足 | 联系超级管理员 |
| 6009 | 操作失败 | 请稍后重试 |
| 6010 | 纠纷不存在 | 检查纠纷ID是否正确 |

### 11.2 异常处理策略

```java
@RestControllerAdvice
public class AdminExceptionHandler {

    @ExceptionHandler(ProductNotFoundException.class)
    public Result<Void> handleProductNotFound(ProductNotFoundException e) {
        return Result.error(6001, "商品不存在");
    }

    @ExceptionHandler(UserNotFoundException.class)
    public Result<Void> handleUserNotFound(UserNotFoundException e) {
        return Result.error(6003, "用户不存在");
    }

    @ExceptionHandler(ConfigNotFoundException.class)
    public Result<Void> handleConfigNotFound(ConfigNotFoundException e) {
        return Result.error(6006, "配置项不存在");
    }

    @ExceptionHandler(PermissionDeniedException.class)
    public Result<Void> handlePermissionDenied(PermissionDeniedException e) {
        return Result.error(6008, "权限不足");
    }
}
```

| 今日统计 | admin:statistics:today | 5分钟 | 今日数据概览 |
| 待审核数量 | admin:audit:pending:count | 1分钟 | 待审核数量 |

### 13.2 数据库优化

- 审核记录表：`product_id`、`admin_id`、`audit_time` 字段加索引
- 系统日志表：`operate_user_id`、`module_name`、`create_time` 字段加索引
- 统计数据表：`stat_date` 字段加唯一索引

### 13.3 异步处理

- 统计数据汇总：通过定时任务异步计算
- 报表导出：异步生成，轮询查询结果
- 批量操作：通过消息队列异步处理
