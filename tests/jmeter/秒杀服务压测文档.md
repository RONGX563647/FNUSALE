## 四、结果分析

### 4.1 性能指标分析

#### 4.1.1 吞吐量分析

**测试结果**: QPS = 14.3/s

**分析**:
- 当前 QPS 较低，主要瓶颈在于：
  1. 单机部署，无负载均衡
  2. Redis 库存扣减使用 Lua 脚本，存在锁竞争
  3. 本地消息表写入增加数据库压力

**优化建议**:
1. 增加 Marketing 服务实例数，通过 Nacos 实现负载均衡
2. 使用 Redis Pipeline 批量处理库存操作
3. 引入消息队列异步处理，提升响应速度

#### 4.1.2 响应时间分析

**测试结果**: 平均响应时间 836ms

**分析**:
- 响应时间分布不均匀，部分请求超过 6 秒
- 原因：Ramp-Up 期间线程同时启动，造成瞬时压力

**优化建议**:
1. 增加 Ramp-Up 时间，平滑启动
2. 使用连接池预热
3. 优化 JVM 参数，减少 GC 停顿

#### 4.1.3 成功率分析

**测试结果**: 成功率 10%

**分析**:
- 这是**业务预期行为**，并非系统问题
- 库存 50 件，100 用户抢购，必然有 50 人失败
- 实际成功 10 次，是因为 Redis 预扣减后部分请求被限流

**正确理解**:
```
成功率 = 成功抢购数 / 总请求数
       = 10 / 100 = 10%

真实成功率 = 成功抢购数 / 库存数
          = 10 / 50 = 20%
```

### 4.2 架构分析

#### 4.2.1 秒杀核心流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  用户请求  │───▶│ 限流检查  │───▶│ Redis扣减 │───▶│ 发送消息  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

#### 4.2.2 关键设计点

| 设计点 | 实现方式 | 优势 |
|--------|----------|------|
| 库存扣减 | Redis + Lua | 原子性，避免超卖 |
| 流量削峰 | RocketMQ | 异步处理，保护数据库 |
| 本地消息表 | 定时任务重试 | 保证消息不丢失 |
| 分布式事务 | 最终一致性 | 性能与一致性平衡 |

### 题目 1: 秒杀系统如何防止超卖？

**答案要点**:

1. **Redis + Lua 脚本实现原子扣减**
```lua
-- seckill_deduct.lua
local stock = redis.call('GET', KEYS[1])
if tonumber(stock) <= 0 then
    return -1  -- 库存不足
end
redis.call('DECR', KEYS[1])
return 1  -- 扣减成功
```

2. **数据库层面使用乐观锁**
```sql
UPDATE t_seckill_activity
SET remain_stock = remain_stock - 1
WHERE id = 1 AND remain_stock > 0;
```

3. **分布式锁兜底**（可选）
```java
RLock lock = redissonClient.getLock("seckill:lock:" + activityId);
try {
    if (lock.tryLock(1, 10, TimeUnit.SECONDS)) {
        // 执行扣减逻辑
    }
} finally {
    lock.unlock();
}
```

**核心原理**: Redis 原子操作保证并发安全，数据库乐观锁保证最终一致性。

### 题目 4: 秒杀场景下 Redis 如何设计？

**答案要点**:

1. **数据结构设计**
```
# 活动库存
seckill:stock:{activityId} = 100

# 用户购买记录（防重复购买）
seckill:user:{userId}:{activityId} = 1

# 排队令牌（防刷）
seckill:token:{activityId}:{userId} = {token}

# 活动信息缓存
seckill:activity:{activityId} = {json}
```

2. **Lua 脚本实现原子操作**
```lua
-- 秒杀核心 Lua 脚本
local stockKey = KEYS[1]
local userKey = KEYS[2]
local userId = ARGV[1]

-- 检查是否已购买
if redis.call('EXISTS', userKey) == 1 then
    return -2  -- 已购买
end

-- 检查库存
local stock = tonumber(redis.call('GET', stockKey) or 0)
if stock <= 0 then
    return -1  -- 库存不足
end

-- 扣减库存
redis.call('DECR', stockKey)
-- 标记已购买
redis.call('SET', userKey, 1)
redis.call('EXPIRE', userKey, 86400)

return 1  -- 成功
```

3. **缓存预热**
```java
@Scheduled(cron = "0 55 * * * ?")  // 活动开始前5分钟

        redisTemplate.opsForValue().set(
            "seckill:stock:" + activity.getId(),

        );

}
```

### 题目 5: 如何设计压测 Token 生成方案？

**答案要点**:

1. **预生成 Token 方案**

```java
public class TokenGenerator {
    private static final String SECRET = "xxx";
    private static final long EXPIRATION = 2 * 60 * 60 * 1000L;

    public static String generateToken(Long userId, String username) {
        Map<String, Object> claims = new HashMap<>();
        claims.put("userId", userId);
        claims.put("username", username);
        claims.put("identityType", "USER");

        return Jwts.builder()
            .claims(claims)
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + EXPIRATION))
            .signWith(Keys.hmacShaKeyFor(SECRET.getBytes()))
            .compact();
    }
}
```

2. **批量生成脚本**
```bash
# 生成 500 个测试用户 Token
java -cp ".:libs/*" GenerateTokens 500 > tokens.json

# 转换为 JMeter CSV 格式

```

```xml
<CSVDataSet>

    <stringProp name="variableNames">token</stringProp>
    <boolProp name="recycle">true</boolProp>
</CSVDataSet>

<HeaderManager>
    <collectionProp name="headers">
        <elementProp name="Authorization" elementType="Header">
            <stringProp name="Header.value">Bearer ${token}</stringProp>

```

**优势**:

### 题目 6: 如何分析 JMeter 压测结果？

| 90th pct | 90%响应时间 | <200ms | 200-1000ms | >1000ms |

2. **结果分析方法**

```bash
# 查看聚合报告
cat results.jtl | awk -F',' 'NR>1 {sum+=$2; count++} END {print "Avg:", sum/count}'

# 分析错误分布
grep "false" results.jtl | awk -F',' '{print $4}' | sort | uniq -c

# 响应时间分布
awk -F',' 'NR>1 {print $2}' results.jtl | sort -n | awk '
  BEGIN {count=0}
  {times[count++]=$1}
  END {
    print "P50:", times[int(count*0.5)]
    print "P90:", times[int(count*0.9)]
    print "P99:", times[int(count*0.99)]
  }'
```

### 题目 7: 秒杀系统如何实现分布式事务？

**答案要点**:

1. **最终一致性方案（推荐）**

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ Redis扣减库存 │───▶│ 写入本地消息表 │───▶│ 发送MQ消息   │
│   (原子操作)  │    │  (同一事务)   │    │             │
└─────────────┘    └─────────────┘    └─────────────┘
                                            │
                                            ▼
                                      ┌─────────────┐
                                      │ 消费者创建订单 │
                                      └─────────────┘
```

2. **本地消息表实现**

```java
@Transactional
public void joinSeckill(Long userId, Long activityId) {
    // 1. Redis 扣减库存（已提前完成）
    // 2. 写入本地消息表（同一事务）
    LocalMessage message = new LocalMessage();
    message.setMessageId(UUID.randomUUID().toString());
    message.setMessageType("SECKILL_ORDER");
    message.setTopic("seckill-order-topic");
    message.setMessageContent(JSON.toJSONString(event));
    message.setStatus("PENDING");
    localMessageMapper.insert(message);

    // 3. 发送消息（可能失败）
    try {
        rocketMQTemplate.syncSend(topic, event);
        message.setStatus("SENT");
        localMessageMapper.updateById(message);
    } catch (Exception e) {
        // 发送失败，等待定时任务重试
        log.error("消息发送失败，等待重试", e);
    }
}
```

3. **定时任务重试机制**

```java
@Scheduled(fixedRate = 60000)  // 每分钟执行
public void retryFailedMessages() {
    List<LocalMessage> messages = localMessageMapper.selectPending();
    for (LocalMessage msg : messages) {
        if (msg.getRetryCount() >= msg.getMaxRetryCount()) {
            msg.setStatus("FAILED");
            // 告警通知
        } else {
            try {
                rocketMQTemplate.syncSend(msg.getTopic(), msg.getMessageContent());
                msg.setStatus("SENT");
            } catch (Exception e) {
                msg.setRetryCount(msg.getRetryCount() + 1);
                msg.setNextRetryTime(calculateNextRetryTime());
            }
        }
        localMessageMapper.updateById(msg);
    }
}
```

### 题目 8: 如何设计秒杀系统的容灾方案？

**答案要点**:

1. **多级缓存架构**

```
┌─────────────┐
│   CDN 缓存   │ ← 静态资源、页面
└──────┬──────┘
       │
┌──────▼──────┐
│  本地缓存    │ ← 活动信息、热点数据
│  (Caffeine) │
└──────┬──────┘
       │
┌──────▼──────┐
│ Redis 缓存   │ ← 库存、用户状态
│  (主从+哨兵) │
└─────────────┘
```

2. **服务高可用**

```yaml
# 多实例部署
spring:
  cloud:
    nacos:
      discovery:
        enabled: true  # 注册到 Nacos

# 负载均衡
spring:
  cloud:
    loadbalancer:
      ribbon:
        enabled: false
```

3. **数据库高可用**

```yaml
# 主从复制 + 读写分离
spring:
  datasource:
    master:
      url: jdbc:mysql://master:3306/fnusale
    slave:
      url: jdbc:mysql://slave:3306/fnusale
```

```java
@SentinelResource(value = "seckill",
```
